# todo

세션 시작 시 자동 표시되는 TODO 파일을 **정해진 포맷 그대로** 편집한다. 표시·아카이브는 번들된 훅 스크립트 `todo-session.py` 가 담당하므로, 이 스킬은 **항목 등록/완료/삭제/조회**만 한다.

## 대상 파일 (단 하나)

- **항상 전역 `~/.claude/todo.md` 한 파일**에만 저장한다. 프로젝트별 별도 파일은 더 이상 쓰지 않는다.
- 프로젝트 구분은 **파일 안의 섹션**으로 한다. 각 항목은 `## 공통` 또는 `## <프로젝트 절대경로>` 섹션 아래에 놓인다.
- `<프로젝트 절대경로>` 는 현재 작업 디렉터리(cwd)의 절대경로다. 워크트리면 워크트리 경로다.

## 파일 포맷 (반드시 유지)

```markdown
# TODO

## 공통

- [ ] 어디서나(모든 프로젝트에서) 보이는 항목

## /Users/me/dev/some-project

- [ ] 그 프로젝트에서만 보이는 미완료 항목
- [x] 그 프로젝트의 완료 항목

## Done (archive)

- [x] 지난 완료 항목  (archived YYYY-MM-DD)
```

- 활성 영역은 `## Done (archive)` 헤더 **위쪽** 전체다. 그 안이 `## 공통` / `## <경로>` 섹션들로 나뉜다.
- 새 항목은 해당 섹션 아래에 `- [ ]` 로 추가한다. 섹션이 없으면 `## Done (archive)` 바로 위에 새 섹션 헤더를 만들고 추가한다.
- 완료 처리는 `- [ ]` 를 `- [x]` 로만 바꾼다. 아카이브 이동은 훅이 세션 시작/종료 시 자동으로 하니 **직접 옮기지 않는다**.
- `## Done (archive)` 헤더와 그 아래 항목은 건드리지 않는다.
- 헤더 없이 제목 바로 아래 떠 있는 항목은 훅이 '공통' 으로 취급한다(구버전 호환). 새로 쓸 때는 명시적으로 `## 공통` 섹션을 쓴다.

## 동작 분기

사용자 의도에서 **귀속 섹션**(공통/현재 프로젝트)과 **액션**을 판단한다. **어떤 액션이든 결과를 보고할 때 어느 섹션에 들어갔는지 반드시 명시한다.**

- 섹션 기본값: **현재 프로젝트**. 별도 언급이 없으면 현재 cwd 의 `## <프로젝트 절대경로>` 섹션에 넣는다(없으면 만든다).
- 공통으로 보내는 경우: 사용자가 "공통", "전역", "어디서나", "프로젝트 무관" 처럼 **프로젝트와 무관함을 명시**했을 때만 `## 공통` 섹션에 넣는다.
- 액션:
  - **추가**: 대상 섹션의 마지막 `- [ ]` 항목 뒤(섹션이 없으면 `## Done (archive)` 바로 위에 헤더와 함께)에 `- [ ]  <할 일>` 추가.
  - **완료/체크**: 해당 항목의 `- [ ]` 를 `- [x]` 로 변경. 사용자가 지목한 텍스트로 매칭하고, 모호하면 후보를 보여주고 되묻는다.
  - **삭제**: 해당 라인을 제거(완료가 아니라 진짜 삭제 요청일 때만). 그 섹션이 비면 헤더도 지운다(공통은 유지).
  - **조회/목록**: 별도 범위 지정이 없으면 **현재 프로젝트 + 공통** 섹션을 보여준다. "전체/모든 프로젝트"라고 하면 파일의 모든 섹션을 보여준다.

## 완료 자동 제안 (확인 후 체크)

사용자가 "완료/체크"라고 말하지 않아도, 진행 중인 작업이 표시된 미완료 항목을 완성하면 **능동적으로 완료를 제안**한다. 단 자동으로 체크하지 않고 **확인을 받은 뒤** 체크한다(오탐으로 인한 오체크 방지).

- **발동 경로**: 이 규칙은 SessionStart 훅(`todo-session.py`)이 매 세션 주입하는 안내문에 포함되어, "todo/투두"를 언급하지 않은 일반 작업 턴에서도 **항상 컨텍스트에 떠 있다**. 그래서 스킬 트리거 없이도 작동한다. (Stop 같은 셸 훅은 작업이 어떤 항목을 완성했는지 의미 판단을 못 하므로 이 용도에 쓰지 않는다.)
- **시점**: 프롬프트로 시작한 작업이 코드/파일 변경을 동반해 완료되는 순간(= "프롬프트 결과마다 커밋"과 같은 시점), 응답을 마치기 전.
- **동작**: "이 작업으로 〈항목〉 이 끝난 것 같습니다. 완료 처리할까요?"처럼 어느 항목인지 짚어 제안하고, 동의를 받은 뒤에만 `- [ ]` → `- [x]`.
- **보수적으로**: 항목을 부분만 진척시켰거나 매칭이 모호하면 제안하지 않거나 후보를 제시해 되묻는다. 확인 없이 체크하지 않는다.

## 절차

1. 전역 파일 `~/.claude/todo.md` 를 대상으로 한다. 없으면 아래 템플릿으로 생성한다:
   ```markdown
   # TODO

   ## Done (archive)
   ```
2. 파일을 Read 한 뒤, 대상 섹션을 찾는다. 현재 프로젝트 섹션 헤더는 cwd 절대경로다(`pwd` 로 확인). 섹션이 없으면 `## Done (archive)` 바로 위에 `## <절대경로>` 헤더와 빈 줄을 만들고 항목을 넣는다.
3. Edit 으로 해당 라인만 수정한다. 섹션 경계와 `## Done (archive)` 경계를 깨뜨리지 않는다.
4. 결과를 한 줄로 보고하되 **어느 섹션인지 항상 포함**한다. 예: `현재 프로젝트 섹션에 추가 (~/.claude/todo.md / ## /Users/me/dev/foo): "API 응답 캐싱"` / `공통 섹션 완료 처리: "주간 회고"`.

## 투두 문맥 파일 (ctx)

투두마다 작업에 필요한 문맥을 별도 파일로 연결해, 나중에 그 투두를 작업할 때 바로 이어갈 수 있게 한다.

- 위치: `~/.claude/todo-context/<영문-슬러그>.md` (전역 단일 파일 체계이므로 ctx 도 전역 한 곳에 모은다).
- 표기: 항목 끝에 `(ctx: todo-context/<슬러그>.md)` 를 붙인다.
  ```markdown
  - [ ] 수식 렌더링 지원 구현하기 (ctx: todo-context/math-rendering.md)
  ```
- 생성 시점: 항목 등록 시 배경/요구사항/관련 파일 같은 문맥이 함께 주어졌거나 대화에서 파악된 문맥이 있으면 만든다. 한 줄로 끝나는 단순 할 일에는 만들지 않는다.
- 내용: 배경/목표, 현재 상태, 관련 파일·링크, 다음 단계를 간결하게. 작성 시점 기준으로 "이 파일만 읽어도 작업을 시작할 수 있는" 수준이면 충분하다.
- 작업 시: 사용자가 해당 투두 작업을 시작하면 ctx 파일을 **먼저 읽고**, 진행되면 내용을 갱신한다.
- 완료/아카이브 후에도 ctx 파일은 지우지 않는다(아카이브 라인의 링크가 기록으로 유지됨).

## 주의

- 항목 텍스트는 사용자가 말한 그대로 적는다. 임의 요약/번역하지 않는다.
- 완료 항목을 아카이브로 직접 옮기지 않는다(훅이 처리). 활성 영역에 `- [x]` 로 두면 다음 세션 시작/종료 때 자동 이동된다.
- 한 번에 여러 항목을 처리할 수 있으면 묶어서 Edit 한다.
- 프로젝트 섹션 헤더는 **절대경로**여야 훅이 cwd 와 매칭해 표시한다. 상대경로/프로젝트명만 쓰면 매칭되지 않는다.

## 자동 표시 훅 (todo-session.py)

이 스킬 폴더에는 세션 시작/종료를 처리하는 훅 스크립트 `todo-session.py` 가 함께 들어 있다. 동기화하면 `~/.claude/skills/todo/todo-session.py` 에 놓인다.

- `start` (SessionStart): 완료 항목을 아카이브로 이동 → **현재 cwd 프로젝트 섹션 + 공통 섹션**의 미완료 항목만 context 로 노출 → 마지막 접속 1시간 초과 시 경과 안내 한 줄 추가.
- `end` (SessionEnd): 완료 항목을 아카이브로 이동.
- 아카이브 시 항목이 모두 빠져 헤더만 남은 프로젝트 섹션은 자동 제거된다(공통 섹션은 유지).
- 마지막 접속 시각은 `~/.claude/.todo-last-access` 에 기록. 임계값은 스크립트 상단 `THRESHOLD_SECONDS`(기본 3600초).

### 훅 등록 (머신별 1회, ao-skills 동기화 대상 아님)

`settings.json` 은 ao-skills 동기화 대상이 아니므로 각 머신의 `~/.claude/settings.json` 에 직접 등록한다.

```json
{
  "hooks": {
    "SessionStart": [
      { "matcher": "startup|resume", "hooks": [
        { "type": "command", "command": "python3 \"$HOME/.claude/skills/todo/todo-session.py\" start" }
      ] }
    ],
    "SessionEnd": [
      { "hooks": [
        { "type": "command", "command": "python3 \"$HOME/.claude/skills/todo/todo-session.py\" end" }
      ] }
    ]
  }
}
```
