# todo

세션 시작 시 자동 표시되는 TODO 파일을 **정해진 포맷 그대로** 편집한다. 표시·아카이브는 번들된 훅 스크립트 `todo-session.py` 가 담당하므로, 이 스킬은 **항목 등록/완료/삭제/조회**만 한다.

## 대상 파일

- 전역: `~/.claude/todo.md` - **기본 대상.** 모든 프로젝트에서 세션 시작 때 보이는 교차 프로젝트 리마인더다. 범위를 따로 말하지 않은 모든 항목은 여기로 간다.
- 프로젝트: `<프로젝트 루트>/.claude/todo.md` - 사용자가 "이 프로젝트만"처럼 명시할 때만 쓰는 현재 프로젝트 전용 목록(git 추적 제외 권장).

`<프로젝트 루트>` 는 현재 작업 디렉터리 기준이다. 워크트리면 워크트리 경로의 `.claude/todo.md` 다.

## 파일 포맷 (반드시 유지)

```markdown
# TODO (...)

- [ ] 미완료 항목
- [x] 완료 항목

## Done (archive)

- [x] 지난 완료 항목  (archived YYYY-MM-DD)
```

- 활성 영역은 `## Done (archive)` 헤더 **위쪽**이다. 새 항목은 여기에 `- [ ]` 로 추가한다.
- 완료 처리는 `- [ ]` 를 `- [x]` 로만 바꾼다. 아카이브 이동은 훅이 세션 시작/종료 시 자동으로 하니 **직접 옮기지 않는다**.
- `## Done (archive)` 헤더와 그 아래 항목은 건드리지 않는다.

## 동작 분기

사용자 의도에서 **범위**(전역/프로젝트)와 **액션**을 판단한다. 모호하면 기본값을 쓰되, **어떤 액션이든 결과를 보고할 때 대상 파일(전역/프로젝트)과 경로를 반드시 명시한다.** 사용자가 "어디에 들어갔는지" 모르는 상태를 만들지 않는다.

- 범위 기본값: **전역**. 별도 언급이 없으면 모든 항목을 전역(`~/.claude/todo.md`)에 넣어 어느 프로젝트에서든 리마인드되게 한다.
- 프로젝트로 보내는 경우: 사용자가 "이 프로젝트만", "프로젝트 todo", "여기서만", "전역 말고" 처럼 **현재 프로젝트 한정 의도를 명시**했을 때만 `<프로젝트 루트>/.claude/todo.md` 로 라우팅한다.
- 액션:
  - **추가**: 활성 영역 마지막 `- [ ]` 항목 뒤(없으면 제목 아래)에 `- [ ]  <할 일>` 추가.
  - **완료/체크**: 해당 항목의 `- [ ]` 를 `- [x]` 로 변경. 사용자가 지목한 텍스트로 항목을 매칭하고, 모호하면 후보를 보여주고 되묻는다.
  - **삭제**: 해당 라인을 제거(완료가 아니라 진짜 삭제 요청일 때만).
  - **조회/목록**: 활성 영역의 `- [ ]`/`- [x]` 를 범위별로 보여준다. 파일을 읽어서 그대로 보여주면 된다.

## 절차

1. 대상 파일 경로를 정한다. 파일이 없으면 아래 템플릿으로 생성한다:
   ```markdown
   # TODO (<전역|프로젝트명>)

   ## Done (archive)
   ```
2. 파일을 Read 한 뒤 Edit 으로 해당 라인만 수정한다. 활성/아카이브 경계와 헤더를 깨뜨리지 않는다.
3. 결과를 한 줄로 보고하되 **대상 파일과 경로를 항상 포함**한다. 예: `프로젝트 todo에 추가 (.claude/todo.md): "API 응답 캐싱"` / `전역 todo 완료 처리 (~/.claude/todo.md): "주간 회고"`.

## 주의

- 항목 텍스트는 사용자가 말한 그대로 적는다. 임의 요약/번역하지 않는다.
- 완료 항목을 아카이브로 직접 옮기지 않는다(훅이 처리). 활성 영역에 `- [x]` 로 두면 다음 세션 시작/종료 때 자동 이동된다.
- 한 번에 여러 항목을 처리할 수 있으면 묶어서 Edit 한다.

## 자동 표시 훅 (todo-session.py)

이 스킬 폴더에는 세션 시작/종료를 처리하는 훅 스크립트 `todo-session.py` 가 함께 들어 있다. 동기화하면 `~/.claude/skills/todo/todo-session.py` 에 놓인다.

- `start` (SessionStart): 완료 항목을 아카이브로 이동 → 미완료 항목을 context 로 노출 → 마지막 접속 1시간 초과 시 경과 안내 한 줄 추가.
- `end` (SessionEnd): 완료 항목을 아카이브로 이동.
- 마지막 접속 시각은 `~/.claude/.todo-last-access` 에 기록. 임계값은 스크립트 상단 `THRESHOLD_SECONDS`(기본 3600초).

### 훅 등록 (머신별 1회, ao-skills 동기화 대상 아님)

`settings.json` 은 ao-skills 동기화 대상이 아니므로 각 머신의 `~/.claude/settings.json` 에 직접 등록한다.

```json
{
  "hooks": {
    "SessionStart": [
      { "matcher": "startup|resume", "hooks": [
        { "type": "command", "command": "python3 \"$HOME/.claude/skills/todo/todo-session.py\" start" }
      ] }
    ],
    "SessionEnd": [
      { "hooks": [
        { "type": "command", "command": "python3 \"$HOME/.claude/skills/todo/todo-session.py\" end" }
      ] }
    ]
  }
}
```
