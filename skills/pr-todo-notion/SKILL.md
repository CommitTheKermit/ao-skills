# pr-todo-notion

GitHub PR의 변경/리뷰 내용을 읽어 Notion 페이지 아래에 체크박스 TODO 페이지를 만든다.
핵심 가치는 **기존 노션 리뷰 TODO 페이지와 똑같이 단순한 체크리스트**를 만드는 것이다. 군더더기(PR 링크 헤더, 메타 줄, 코멘트 링크, 파생 섹션, 후속 작업 섹션)를 넣지 않는다.

## 산출물 형식 (고정)

기존 노션 "리뷰 TODO" 페이지와 100% 동일한 형식을 따른다.

```markdown
## <N>차 리뷰 TODO (<리뷰 날짜>, <리뷰 상태>)
- [ ] <해야 할 일(행동 중심)> (<위치: 파일명 또는 보고서 섹션>)
	- 세부 근거/체크포인트
	- (필요하면 한 줄 더)
```

- 맨 위에 `## <N>차 리뷰 TODO (날짜, 상태)` 헤더 한 줄만 둔다. PR 링크/팀/리뷰어 같은 메타 줄을 넣지 않는다.
- 각 항목은 `- [ ] 할 일 (위치)` 형태. 위치는 괄호 안에 파일명(코드 PR) 또는 보고서 섹션명(문서 PR)을 적는다.
- **코멘트 링크(html_url)를 달지 않는다.** 추적 링크 없이 위치 표기만 한다.
- 파생 섹션("본문 기반 파생 TODO")이나 "후속 작업" 섹션을 따로 만들지 않는다. 코멘트에 직접 매핑되지 않는 항목도 같은 리스트 안에 자연스럽게 섞어 둔다.
- 리뷰어가 남긴 코멘트는 하나도 누락하지 않는다. 모든 코멘트가 최소 한 개의 체크박스에 반영돼야 한다.

## 워크플로우

```
[1] PR 데이터 수집  →  [2] 대상 페이지 + 형식 확정  →  [3] TODO 하위 페이지 생성  →  [4] 보고
```

### 1. PR 데이터 수집

`gh`로 PR 본문과 리뷰 코멘트를 가져온다. 코멘트는 빠질 수 있으므로 세 종류를 모두 조회한다. (링크는 산출물에 쓰지 않지만, 코멘트 내용과 리뷰 날짜/상태는 필요하다.)

```bash
# 본문 / 변경 파일 / 리뷰 상태·날짜
gh pr view <PR번호> --repo <owner/repo> --json title,body,files,reviews

# 라인 단위 리뷰 코멘트 (각 TODO의 근거)
gh api repos/<owner/repo>/pulls/<PR번호>/comments --jq '.[] | {path, line, body}'

# PR 전체 코멘트(이슈 코멘트)
gh api repos/<owner/repo>/issues/<PR번호>/comments --jq '.[] | {body}'

# 리뷰 상태/제출일 (헤더의 날짜·상태에 사용)
gh api repos/<owner/repo>/pulls/<PR번호>/reviews --jq '.[] | {state, submitted_at, body}'
```

헤더의 `<리뷰 날짜>`는 리뷰 제출일(`submitted_at`), `<리뷰 상태>`는 `state`(CHANGES_REQUESTED 등)를 쓴다.

### 2. 대상 페이지 + 형식 확정

- 사용자가 페이지를 지정하면 `Notion search`로 찾는다. 지정이 없으면 "리뷰 TODO 리스트 모음" 페이지를 검색해 부모로 삼는다.
- **기존 리뷰 TODO 페이지 하나를 `Notion fetch`로 열어 실제 형식을 확인하고 그 형식을 그대로 따른다.** (위 "산출물 형식"이 그 기준이다.)
- 새 TODO는 모음 페이지의 **하위 페이지**(`parent: {type:"page_id", page_id: ...}`)로 만든다.
- 코드 PR이면 위치에 파일명을, 문서/기획 PR이면 보고서 섹션명을 적는다. 본문의 미해결 사항(합의 안 된 가정, 검증 가설)도 같은 리스트에 체크박스로 추가한다.

### 3. TODO 하위 페이지 생성

`Notion notion-create-pages`로 하위 페이지를 만든다.

- icon: 주제에 맞는 이모지 1개
- title: `PR #<번호> <한줄요약> 리뷰 TODO (<팀/리포>)` 형태
- content: 위 "산출물 형식"만. 헤더 한 줄 + 체크박스 목록. 그 외 블록 금지.

### 4. 보고

생성된 노션 페이지 URL과 반영한 코멘트 개수를 한 번에 보고한다.

## 절대 어기지 말 것

- 산출물에 PR 링크 헤더, 메타 줄, 코멘트 링크, 파생/후속 섹션을 넣지 않는다. 기존 노션 형식을 벗어나지 않는다.
- 사용자가 다른 형식을 지정하면 그것을 우선한다. (전역 규칙: UI/형식 결정 전 사용자에게 알릴 것)
- 리뷰 코멘트를 누락하지 않는다. 모든 코멘트가 체크박스에 반영돼야 한다.
- 노션 작업만 한 경우 코드 변경이 없으므로 git 커밋하지 않는다.

## 필요 도구

- `gh` CLI (PR/코멘트 조회)
- Notion MCP: `search`, `fetch`, `notion-create-pages`, `notion-update-page`
