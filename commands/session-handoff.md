긴 작업 세션을 새 세션으로 끊어 갈 때, 다음 세션의 에이전트가 **추가 탐색 없이 바로 이어서 작업**할 수 있도록 문맥을 압축해 넘긴다.

`$ARGUMENTS`로 저장 경로나 강조할 범위가 지정되면 그에 따른다. 비어 있으면 현재 작업 세션 전체를 대상으로 한다.

## 산출물 두 가지

1. **핸드오프 프롬프트** - 새 세션 첫 메시지로 사용자가 그대로 붙여넣을 마크다운 블록. 채팅에 출력한다.
2. **handoff.md (+ 아카이브)** - 저장소 루트(또는 사용자가 지정한 경로)의 `handoff.md`에 같은 내용을 저장하고, 동시에 `handoff/handoff_yyMMdd_HHmm.md`로도 아카이브한다. 새 세션이 파일로도 참조 가능.

두 산출물의 내용은 동일하다. 프롬프트는 "붙여넣기용", 파일은 "저장소에 남기기용".

### 파일 네이밍 / 배치 규칙

- **최신 진입점은 항상 루트 `handoff.md` 하나로 고정**한다(덮어쓰기). Claude Code의 `@` 파일 멘션 정렬 기준은 문서화되어 있지 않아 "최신이 목록 맨 위"를 보장할 수 없으므로, 정렬에 의존하지 않고 고정 파일명으로 최신 진입점을 단일화한다. 새 세션은 `@handoff` 로 늘 이 파일을 집는다.
- **이력 보존용 아카이브**는 `handoff/handoff_yyMMdd_HHmm.md` 로 남긴다. **포맷은 `yy`(2자리 연)+`MM`+`dd`+`_`+`HH`+`mm`**, 24시간제, 콜론 없음(macOS Finder/쉘 경로 안전).
- 타임스탬프는 지어내지 말고 `date "+%y%m%d_%H%M"` 로 실제 시각을 구해 파일명에 쓴다.

## 핵심 원칙

- **사실만 적는다.** 추측/희망사항이 아니라 이번 세션에서 실제로 확정/구현/검증된 것만. 미검증은 "미검증"이라고 명시.
- **다음 세션이 헤맬 지점을 미리 막는다.** 재사용할 코어, 건드리지 말 것, 이미 내린 결정과 그 이유를 적어 같은 논의를 반복하지 않게 한다.
- **경로/명령은 복붙 가능하게.** 절대경로, 정확한 스크립트명, 실제 커밋 규칙.
- **간결하게.** 한 화면에 핵심이 들어오도록. 코드 전문 붙여넣기 금지, 파일 경로로 가리킨다.

## 작성 절차

### 1. 현재 세션에서 정보 수집
다음을 실제로 확인한다(모르면 사용자에게 묻거나 명령으로 확인, 지어내지 않는다).

- 프로젝트 절대경로, 현재 git 브랜치, 커밋 규칙(co-author 라인, em dash 금지 등)
- 이번 세션에서 **확정한 결정**과 그 **이유** (가장 중요. 방향 전환이 있었다면 전/후와 근거)
- 이번 세션에서 **구현/변경한 것** (커밋된 것 / 워킹트리만인 것 구분)
- 빌드/테스트 상태(통과 개수 등), 설치된 의존성/버전
- **다음에 할 일**을 실행 가능한 단계로 분해
- 재사용할 코어 모듈, 건드리면 안 되는 것
- 제약/함정(미검증 영역, 환경 요구사항 등)

### 2. 템플릿으로 작성
아래 템플릿을 채운다. 해당 없는 섹션은 생략 가능하나, "확정한 결정"과 "다음에 할 일"은 반드시 채운다.

```markdown
# <프로젝트/작업 한 줄 제목>

## 프로젝트 위치 / 브랜치
- 경로: `<절대경로>`
- 브랜치: `<브랜치>` (<main 병합 여부>)
- 커밋 규칙: <한 프롬프트=한 커밋, em dash 금지, co-author 라인 등>

## 무엇을 만드는가
<목적과 핵심 메커니즘 2~4줄>

## 이번 세션에서 확정한 결정 (중요)
<결정 한 줄> + 이유.
방향 전환이 있었다면: 기존 방식 -> 새 방식, 왜 바꿨는지, 검증된 사실.

## 재사용할 코어 / 건드리지 말 것
- `<경로>` - <역할, 왜 재사용/보존>

## 현재 상태
- 빌드: <통과 여부>
- 테스트: <통과 개수/파일>
- 의존성: <새로 설치한 것 + 버전>

## 다음에 할 일 (구현 단계)
1. <파일/모듈> - <무엇을>
2. ...
(실행 가능한 단위로. 입력 스키마/시그니처 등 결정된 세부는 함께 적는다)

## 제약 / 함정
- <미검증 영역, 환경 요구, 주의점>
```

### 3. 두 산출물 출력
- 채팅에 핸드오프 프롬프트를 코드펜스 안에 출력해 사용자가 복사하기 쉽게 한다.
- 같은 내용을 두 위치에 Write 한다(경로를 사용자가 지정하면 그 경로 기준):
  - 루트 `handoff.md` - 최신 진입점(고정 파일명). 덮어쓰기 한다(매번 최신으로 갱신되는 것이 의도이므로 덮어쓰기 전 확인 불필요).
  - `handoff/handoff_yyMMdd_HHmm.md` - 타임스탬프 아카이브. `date "+%y%m%d_%H%M"` 로 실제 시각을 구해 파일명을 만든다. `handoff/` 디렉토리가 없으면 만든다.

## 주의

- handoff.md / handoff/ 커밋 여부는 사용자에게 맡긴다(자동 커밋하지 않는다). 단 프로젝트의 "프롬프트 작업 완료 시 커밋" 규칙이 있으면 그에 따른다.
- 비밀값(토큰/키)은 핸드오프에 넣지 않는다. 필요하면 "`.env`의 X를 설정" 식으로 가리킨다.
